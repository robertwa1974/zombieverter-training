C01 — CHIP IDENTIFICATION

Know Your ChipBefore touching firmware — identify the MCU

ChipMarkingsFirmware
STM32F107ST logo + STM32F107Main branch — all current features
GD32F107GD logo + GD32F107GD_Zombie branch — abandoned, diverged
🔴 GD Chip Warning

GD32F107 boards were produced during the 2021–2022 chip shortage. The separate GD firmware branch was abandoned shortly after and has substantially diverged from main. GD boards cannot run current firmware and miss all recent features.

How to check

Look at the main IC on the board under good lighting. The manufacturer logo (ST vs GD) and chip number are printed on the chip surface.

C01 — METHOD 1

Web UART UpdateFor routine updates on a working board

1
Download firmware .bin
From GitHub releases page. Rename to stm32_vcu.bin if needed.
2
Connect to VCU Wi-Fi
Connect to ZombieVerter access point, navigate to 192.168.4.1
3
Find UART Update field
In the web interface — select your .bin file
4
Upload and wait
Do NOT power-cycle mid-update. Wait for completion.
5
Verify firmware version
Reconnect to web interface. Firmware version should show in top bar. If it shows "null" — use ST-Link recovery.

After Any Update

🔴 Always do these steps

1. Reselect inverter type in Parameters → Comms
2. Save to flash
3. Reboot VCU
Parameters may not migrate correctly between major versions.

Windows 10 Issue

⚠ Known reliability problem

Windows 10 has known issues with the UART update method. Consider using Ubuntu (WSL2 works) or switching to ST-Link method if UART update fails repeatedly.

Serial terminal (if needed)

Baud: 115200 · Data: 8 · Parity: None · Stop bits: 2 (unusual — most tools default to 1)
