F02 — WHY IT EXISTS

The ProblemSalvaged EV parts don't talk to each other

The situation

You've salvaged a Nissan Leaf motor and inverter from a wreck. You want to put it in your BMW. The inverter was designed to talk to a Leaf BMS, a Leaf brake controller, a Leaf dashboard — none of which you have.

Every donor component speaks its own proprietary protocol. Building a custom controller for every possible combination would take months per build.

The ZombieVerter Solution

A general-purpose VCU with pre-built support for the most common donor components. It's the translator between your salvaged EV parts and your conversion project.

What it coordinates

# ZombieVerter sits in the middle Throttle pedal → VCU → Inverter torque cmd Brake signal → VCU → Regen enable ISA shunt → VCU → Pack voltage/current BMS → VCU → Current limits Charger → VCU → Charge control Ignition/Start → VCU → Contactor sequence
No coding required

All configuration through a browser-based web interface. No laptop software, no USB cable, no command line.

F02 — SYSTEM OVERVIEW

How It Fits InThe VCU is the central coordinator

# A typical ZombieVerter-based conversion [Throttle pedal] ──── analog ───────────────────────────────────────────────┐ [Brake signal] ──── digital ──────────────────────────────────────────────┤ [Fwd/Rev switch] ──── digital ──────────────────────────────────────────────┤ [Ignition/Start] ──── digital ──────────────────────────────────────────────▼ ┌─────────────────┐ [ISA Shunt] ──────────── CAN ──────────────────────────────▶│ │──── CAN ──▶ [Inverter] [BMS] ──────────── CAN ──────────────────────────────▶│ ZombieVerter │──── CAN ──▶ [Charger] [BMW i3 LIM] ──────────── CAN ──────────────────────────────▶│ VCU │──── LIN ──▶ [Heater] │ │── Serial ▶ [GS450h] └────────┬────────┘ │ low-side outputs [Neg Contactor] [Pos Contactor] [Precharge]
Key principle

The VCU reads driver inputs, monitors pack state via the shunt, sequences contactors, and sends torque commands to the inverter — all configurable via browser, no code.

Cell-level BMS note

ZombieVerter uses the ISA shunt for pack-level monitoring. Cell-level functions (balancing, per-cell voltage) are handled by a separate BMS like SimpBMS, which talks to the VCU via CAN.

# 6 pedal wires → 4 VCU pins Ch1 GND ─┬─── Pin 45 Ch2 GND ─┘ Ch1 5V ─┬─── Pin 48 Ch2 5V ─┘ Ch1 sig ──── Pin 47 Ch2 sig ──── Pin 46

Single vs Dual Channel

ModeRedundancyRisk if fails high
Dual (potmode=dual)Yes — VCU detects mismatchFault → torque cut
Single (potmode=single)NoneUnintended full throttle
🔴 Use dual channel

If the single pot fails in the high direction with no redundancy, the result is uncontrolled acceleration. All modern drive-by-wire pedals have two sensors — use both.

⚠ Ground rule

Throttle grounds (Pin 45) connect to the VCU — NOT to chassis ground. Erratic potnom is often a bad ground here.

W02 — CRITICAL CONCEPT

The ZV Pot RangeOpposite to almost every other system

How most systems work

Set potmin/potmax outside mechanical travel. If value goes beyond limits → fault → safe cutoff.

Mechanical: 400→2600
potmin = 300 / potmax = 2700
🔴 ZombieVerter is OPPOSITE

Set potmin/potmax inside the mechanical travel range. The pedal physically travels past both limits.

ZombieVerter approach

# Watch 'pot' spot value while moving pedal At rest: ADC = 400 Full travel: ADC = 2600 potmin = 450 ← just ABOVE resting potmax = 2550 ← just BELOW full travel # Result: potnom = 0 at rest, 100 at WOT
⚠ Common mistakes

potmin too low → potnom never reaches 0 at rest → phantom throttle input

potmax too high → potnom never reaches 100 → can't get full torque

TRACK W — WIRING · MODULE W03

Contactor &
Precharge

The most safety-critical part of the installation. Getting this wrong can weld contactors permanently or block run mode indefinitely.

🔴 Safety critical

Read this module completely before wiring. Incorrect contactor wiring can cause violent inrush current, welded contactors, or unexpected HV live conditions.

⚠ Most common first-start failure

UDC = 0 or udcsw set too high. The VCU cannot precharge without a valid UDC reading. This is #1 cause of "stuck in precharge".

Prerequisites: H01, F03 · V2.30A
W03 — CONTACTOR SYSTEM

Three ContactorsEach has a specific purpose and sequence

ContactorPurposeVCU Pin
NegativeConnects battery negative to HV ground — always firstPin 31
PrechargeHV positive through resistor — limits inrush currentPin 34
Main (positive)Direct battery positive to HV bus once precharge donePin 33

Sequence

START signal received → Negative contactor CLOSES (always first) → Precharge contactor CLOSES (resistor limits current) → UDC rises toward pack voltage → UDC ≥ udcsw ? YES → Main contactor CLOSES Precharge contactor OPENS → opmode = "run" NO → timeout → PRECHARGE FAULT

Wiring Rule — Low-Side Only

🔴 Critical rule

All contactor outputs are low-side switches. They pull the GROUND leg of the coil to ground. The POSITIVE leg of every coil must connect to 12V+. Never apply 12V+ to these output pins.

# Contactor coil wiring 12V+ ──────── [Coil +] [Coil −] ──── VCU Pin (31/33/34) GND ──────── VCU internal
⚠ Polarity

Some contactors are polarity-sensitive. Check the datasheet — backwards wiring reduces hold force and life.

W03 — UDC

UDC — Pack VoltageThe VCU cannot precharge without a valid reading

What is UDC?

UDC = the VCU's measurement of HV bus voltage. It drives the entire contactor sequencing logic. If UDC = 0, precharge fails immediately every time.

UDC Sources (shuntType)

shuntTypeSource
ISAISA IVT shunt (most common)
SBOXBMW S-BOX
VAGVW/Audi battery system
LEAFNissan Leaf inverter CAN
🔴 Must match hardware

If shuntType is set to a source that isn't present or communicating, UDC = 0 and precharge will always fail.

ISA Shunt — Permanent Power Rule

🔴 ISA shunt must have permanent 12V

Same always-on supply as the VCU. If powered from ignition-switched supply, the shunt loses UDC when key is off. Next startup → stale/zero UDC → precharge fails.

ISA Shunt Position

# ISA U1 must measure INVERTER side # NOT battery side of main contactor Battery+ ── [Main contactor] ──────── HV Bus+ │ [Precharge] ── [R] ──────────┤ ↑ ISA U1 here
⚠ Common U1 mistake

U1 on battery side = reads full pack voltage immediately = main contactor fires before caps are charged = potentially welded contacts.

W03 — SETUP

ISA Init & Precharge ResistorOne-time setup and component sizing

ISA Shunt Initialisation

New ISA shunts must be initialised once before they communicate on CAN.

1
Wire permanent 12V + CAN bus
Same always-on 12V feed as VCU
2
Set shuntCan in Parameters → Comms
Match the CAN bus the shunt is physically connected to
3
Set ISAMode = Init, save to flash
4
Power cycle both VCU and shunt simultaneously
5
Set ISAMode = Normal, save, reboot
If init fails

Power shunt first, wait 2–3 seconds, then power VCU. This lets shunt fully boot before VCU tries to communicate.

Precharge Resistor Sizing

SpecValue
Typical range40Ω – 100Ω
Power rating50W – 100W
Common starting point100Ω / 100W

Sizing logic

Higher R = slower precharge, more protection, less stress on contactors — but must complete within 5 seconds.

Lower R = faster precharge, but less protection against inrush current damage.

W06 — IGNITION & START

Ignition ON & StartTwo separate signals — sustained vs momentary

Ignition ON — Pin 15

PinSignal typeWhen
Pin 15Sustained 12VKey in ON position — stays high while driving

Tells VCU the vehicle is "on" but does NOT start contactor sequencing by itself.

Bench test

Toggle switch between 12V+ and Pin 15. Toggle on = key to ON position.

Start — Pin 52

PinSignal typeDuration
Pin 52Momentary 12V pulse≥500ms — like turning a key to START
⚠ Momentary only

The start signal must be momentary — NOT sustained. Wiring it as a sustained signal can cause issues re-triggering precharge after a fault.

The Inverter Enable Relay (Pin 32)

VCU controls a relay via Pin 32 to provide managed "ignition" to the inverter, charger, and DC-DC. These devices should NOT be powered until the VCU is ready.

# Inverter enable relay wiring 12V+ ─── [Relay coil +] [Relay coil −] ─── Pin 32 (VCU) 12V+ ─── [Relay switch COM] [Relay switch NO] ─── Inverter enable └── Charger enable └── DC-DC enable
The "Holy Trinity" for run mode

Pin 56 (permanent 12V) + Pin 55 (ground) + Pin 15 (ignition ON) + Pin 52 (start pulse) + Pin 54 or 53 (direction) = minimum for run mode

I03-X — HIGH VOLTAGE CONNECTION

The HV BypassThe most expensive mistake in GS450h builds

Two connection points — only one is correct

The GS450h inverter has two places you can connect your HV battery. One is the obvious OEM terminal. The other is a set of brass pillars hidden under a plastic cover. These are not interchangeable.

🔴 OEM Terminals — DO NOT USE

Connecting HV to the OEM positive/negative terminals routes current through the boost inductor and boost IGBT. These components are rated for ~100A maximum and are not designed for continuous EV traction loads. Everything works fine on the bench and at low speed. At highway speed under load — the inverter destroys itself. At least two confirmed inverter failures from this mistake.

✓ Brass Pillars — Correct Connection

Two brass pillars accessed by removing a small black plastic cover and two M6 bolts (10mm head). These connect HV directly to the motor drive stage, bypassing the boost converter entirely. This is the correct connection for EV use.

How to Access the Brass Pillars

1
Locate the black plastic cover
On the inverter body — not the main HV cable entry points. Look for a small rectangular cover near the HV bus.
2
Gently prize off the cover
Toyota uses adhesive. Prize gently — don't crack it. Some builders 3D print a replacement cover.
3
Remove two M6 bolts (10mm head)
Two small bolts secure the brass pillars. Remove them to expose the pillar threads.
4
Connect HV+ to one pillar, HV− to the other
Polarity matters. HV+ to positive pillar, HV− to negative pillar. Secure with appropriate lugs and bolts.
⚠ Bench test tip

For bench testing, a 60V 5A power supply into the brass pillars is sufficient to verify sync serial communication and rotation at low speed. You do not need pack voltage to confirm the system works.

I03-X — OIL PUMP SYSTEM

The Oil PumpNot optional — the inverter overheats without it

Why the Oil Pump Exists

The GS450h inverter has its own internal oil cooling circuit. In the OEM hybrid vehicle, this pump is driven mechanically. In an EV conversion, there is no engine — so an electric oil pump must run continuously while the inverter is active.

🔴 Run it from precharge. Always.

If the oil pump is not running when the inverter is active, the inverter will overheat. This is not a long-term problem — it's a short-term one. The pump must spin up during precharge and remain on throughout run mode.

Input Shaft Lock — The Critical Mod

Without locking the input shaft, MG1 can only spin the engine input shaft — it cannot send torque to the output. The lock forces MG1 torque through the power split device to the output shaft, giving you both motors driving the wheels.

🔴 Without the lock

MG1 provides no useful traction — it just spins freely. You lose roughly half your available torque and power. This is a mechanical modification, not a software setting.

What the lock looks like

A plate or bar with a spline fitting that bolts to the bellhousing flange, engaging the engine input shaft splines and preventing rotation. For bench testing: Damien uses a vice grip and cable tie. For a real vehicle: a purpose-made billet lock plate.

⚠ MG1 torque ratio

With the shaft locked, MG1 torque is geared up to MG2 at approximately 2.6–2.7:1. This means MG1 effectively contributes more torque at the output than its nameplate rating suggests — it is a significant contribution, not a minor addition.

The EVSE generates CP. It signals available current and confirms the car is ready to charge. The car pulls down the pilot voltage through a resistor to signal "ready to charge" (State C).

Control Pilot (CP)

The EVSE generates CP. It signals available current and confirms the car is ready to charge. The car pulls down the pilot voltage through a resistor to signal "ready to charge" (State C).

Assign CpSpoof to a VCU PWM output pin. VCU generates a State C CP signal so chargers that need it (Outlander, Foccci, some VW) confirm the car is ready — without a full J1772 charge port.

PP detects physical cable insertion. A resistor inside the cable plug pulls down a 5V sense line. Different cable current ratings use different resistor values.

A05 — CAN BUS WIRING

CAN Bus Physical WiringTermination and topology — the rules that actually matter

Termination — The Most Skipped Step

A CAN bus must have exactly two 120Ω termination resistors — one at each physical end of the bus. Without correct termination you get signal reflections, corrupted frames, and intermittent communication failures that are very hard to diagnose.

🔴 Exactly two 120Ω resistors — one at each end

One resistor at the VCU end (often built into the VCU). One resistor at the furthest device on that bus. Not three, not one — two. Between CAN H and CAN L.

⚠ Check before assuming

Some inverters and OBCs have built-in 120Ω termination. If they do and you add an external resistor, you end up with too many. Check your device datasheet or measure with a multimeter across CAN H/L with power off — should read ~60Ω (two 120Ω resistors in parallel).

Topology — Bus Not Star

CAN is a bus topology. All devices connect to the same two wires (CAN H and CAN L) running from one end to the other. Avoid star topologies and long stub branches.

# Correct — bus topology [120Ω] ─── VCU ─── ISA Shunt ─── Inverter ─── [120Ω] # Wrong — star topology VCU ├── ISA Shunt ├── Inverter ← signal reflections └── BMS
Cable recommendations

Twisted pair is strongly preferred for CAN H/CAN L — ideally shielded twisted pair for runs near HV cables or in noisy environments. Unshielded twisted pair works for most bench setups and short in-vehicle runs. Avoid running CAN wiring parallel to HV cables over long distances.

CAN1/CAN2 label swap

There is a known documentation issue where CAN1 and CAN2 are labelled in reverse in some older wiring diagrams. If a device won't communicate despite correct wiring, swap the inverterCan or shuntCan parameter value before rewiring.

# Common starting configuration CAN1 @ 500 kbps ├─ Traction inverter (inverterCan = CAN1) └─ BMS / SimpBMS (bmsCan = CAN1) CAN2 @ 1000 kbps └─ ISA shunt (shuntCan = CAN2) CAN3 @ 500 kbps fault-tolerant └─ CHAdeMO socket (6 PCB jumpers) # If VW OBC: needs BOTH CAN1 and CAN2 # If Leaf PDM: same bus as inverter is fine

No comms after correct wiring? Check in order: correct bus assignment parameter · correct baud rate · termination (measure ~60Ω across H/L with power off) · try swapping CAN1/CAN2 assignment · check twisted pair isn't reversed (H and L swapped).

Fuel Gauge — Digipot Method

For vehicles without CAN fuel gauge control, a digital potentiometer (digipot) IC can be wired between the VCU and the OEM fuel gauge sender wire. The VCU controls the digipot resistance, which moves the fuel needle to show state of charge.

How it works

The OEM fuel gauge reads a variable resistance from the tank sender. A digipot replaces the sender — the VCU sends step commands (e.g. "6 steps, 5 steps") to set the resistance, which moves the needle. Tune the step values by trial: set steps, watch needle, adjust until needle matches SoC accurately at key reference points (full, half, empty).

⚠ Fuel light threshold

The fuel warning light triggers when the gauge reads low. With a digipot, setting steps = 0 will trigger the low fuel warning — useful as a low-SoC alert. Test this deliberately before relying on it.

Temperature gauge

Some vehicle classes also allow the temperature gauge to be driven from the inverter heatsink temperature (temphs) or motor temperature (tempm). Useful as an overtemperature warning.

Path 2 — Fork for Your Own Use

Fork the repo, clone locally, make your changes, compile, flash. You're entirely in control. No obligation to contribute back. Your fork runs independently of the main release cycle.

# Clone with all submodules — critical git clone --recurse-submodules git@github.com:damienmaguire/Stm32-vcu.git # libopeninv and other deps are submodules # Without --recurse-submodules the build fails
⚠ GD32 boards cannot run current firmware

If you have a GD32F107 board, the GD_Zombie branch was abandoned and has substantially diverged. Self-building for a GD board means starting from that old branch — you will not have recent features. See C01.

Build environment

The openinverter firmware uses an ARM GCC toolchain. Build documentation is on the openinverter.org wiki. Johannes Huebner's libopeninv is the underlying framework — his GitHub is the reference for the parameter system and CAN class architecture.

Path 3 — Pull Request to Main Branch

If you implement something useful for the community, you can submit a pull request. This is how the BMW E31, OBD2, analog IO, and most new inverter support landed in the firmware — contributions from community members.

The Testing Requirement

🔴 Changes must be tested in real vehicles before merging

This is non-negotiable. The firmware runs HV systems in moving vehicles. A bug that gets through to a release can cause a precharge failure at speed, unintended motor torque, or a contactor fault. Damien personally tests new features across multiple vehicles — typically 3–4 different builds — before any release goes out.

What good contributions include

Working code · Evidence of testing in at least one real vehicle · CAN log data if it's a new device integration · A description of what was tested and what edge cases were considered. The community is welcoming to well-tested contributions — this is how the project grows.
