C06 — OPMODE

Operating ModesThe normal startup sequence: off → precharge → run

opmodeMeaning
offVCU powered but ignition not on, or in standby
prechargeStart signal received — contactors sequencing
pch failPrecharge sequence timed out — something blocked it
run✓ Precharge complete, main contactor closed, torque available
chargeCharge mode active — HV contactors closed for charging
Normal startup

off → precharge → run

Failed startup

off → precharge → off (or pch fail)
Something prevented precharge from completing. Check status field immediately.

Status Field Messages

status showsMeaningFix
UDC below udcswPack voltage hasn't reached precharge thresholdLower udcsw. For bench: set to 0–15V
Throttle 1 2Throttle channel fault — mismatch between channelsCheck pot2 calibration, check wiring
din_break onBrake signal is activeCheck brake switch type (should be NO)
⚠ Note on "lastError"

The lastError field is not always actually an error — it records the last significant status change. Think of it as "last known status" rather than "last fault."

C06 — ROOT CAUSE #1

udcsw — The #1 First-Start FailureDefault 330V blocks all bench testing

330V
udcsw default value
= precharge completion threshold
= bench test with no HV will NEVER reach this
🔴 Why this is intentional

Setting udcsw too low is dangerous — the main contactor could close before inverter capacitors are adequately charged, potentially destroying the inverter. 330V is a conservative safe default for common Leaf/Prius systems.

Diagnosis Procedure

1
Open Spot Values, enable Auto Refresh
2
Ignition ON → press Start
3
Watch opmode go to "precharge"
If it does, good — sequence started
4
Watch status field
"UDC below udcsw" → udcsw is the problem
5
Go to Parameters → Contactors → udcsw
6
Set to 0 (bench) or 95% of pack voltage
Save to flash. Try again.
SituationSet udcsw to
Bench, no HV0–5V
Bench, low-voltage sourceJust below source voltage
Real vehicle~95% of nominal pack voltage
C06 — ROOT CAUSES #2 & #3

Brake & Throttle FaultsRun mode but no torque — the confusing ones

din_brake stuck "on"

The VCU is in run mode. Throttle potnom changes correctly. But the motor does nothing. This is the brake interlock.

🔴 Must be "off" for any torque

din_brake = "on" → zero torque output, regardless of throttle position or opmode.

CauseFix
Brake wired to constant 12VCheck brake switch wiring
NC brake switch instead of NOUse the normally-open terminal
Pin 49 floating, picking up noiseAdd 10kΩ pull-down to ground
Brake pedal physically held downRelease the brake pedal

Throttle 1 2 Fault

VCU has a problem with one or both throttle channels. Appears in status field in run mode.

1
Check both pot AND pot2 in Spot Values
Both should change when pedal moves. If one is fixed → wiring fault on that channel.
2
Check pot2min and pot2max
If still at defaults (0/4095) while potmode=dual → plausibility fault guaranteed. Calibrate both channels.
3
Check potmode parameter
If only one channel wired, set potmode=single
